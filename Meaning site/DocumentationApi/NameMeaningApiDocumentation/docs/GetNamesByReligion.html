GET /api/religion/{religion}

Retrieve paginated names from our 65,000+ database filtered by religion (Islamic, Christian, Hindu) with advanced filters, sorting, and ISR support.

Description

Access our comprehensive database of over 65,000 religious names with detailed metadata. This endpoint returns paginated results with comprehensive name information including meanings, origins, cultural context, and more. Optimized for Next.js ISR with 60-second revalidation for optimal performance.

Path Parameters

religion string required

The religion category to filter names by.

islamic christian hindu

Query Parameters

page integer optional
default: 1

Current page number for pagination.

limit integer optional
default: 40

Number of names per page (min: 1, max: 100).

sort enum optional
default: asc

Sort order for results.

asc desc popularity
gender enum optional

Filter by gender.

boy girl
letter string optional

Filter by starting letter (A-Z).

origin string optional

Filter by name origin (e.g., Arabic, Hebrew, Sanskrit).

Request Examples

Basic request with defaults

GET https://namverse-api.vercel.app/api/religion/islamic?page=1&limit=40&sort=asc

Filter by gender

GET https://namverse-api.vercel.app/api/religion/christian?gender=girl&sort=popularity

Filter by letter and origin

GET https://namverse-api.vercel.app/api/religion/hindu?letter=A&origin=Sanskrit&limit=20

Next.js Integration

// app/religion/[religion]/page.js

export const revalidate = 60; // ISR: Revalidate every 60 seconds

const API_BASE = 'https://namverse-api.vercel.app/api/religion';

export async function generateStaticParams() {
  return ['islamic', 'christian', 'hindu'].map((religion) => ({ 
    religion 
  }));
}

export default async function ReligionPage({ params }) {
  const { religion } = params;
  
  const response = await fetch(
    `${API_BASE}/${religion}?page=1&limit=40`,
    { next: { revalidate: 60 } }
  );
  
  const data = await response.json();
  
  return <ReligionBrowser initialData={data} />;
}
